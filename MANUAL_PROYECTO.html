1Estructura y Arquitectura

Arquitectura General

El proyecto implementa una arquitectura por capas limpia que separa claramente las responsabilidades en tres niveles:

🗄️ Data Layer Acceso a datos, servicios y configuración
🎯 Domain Layer Modelos de datos puros e independientes
🎨 Presentation Layer Interfaz de usuario y pantallas

2Componentes Principales

Gestión de Estado y Dependencias

Provider (^6.1.2): Sistema de inyección de dependencias y gestión reactiva de estado. Se utiliza para:
  • Providores de servicios (Auth, Cars, Reports)
  • Acceso global a datos de aplicación
  • Notificación de cambios a widgets
BLoC (^8.1.5): Para lógica de negocio compleja, especialmente:
  • GpsBloc: Gestión de ubicación en tiempo real
  • Manejo de eventos y estados
  • Separación clara de lógica de presentación

Servicios Críticos

Servicio Responsabilidad Ubicación
AuthService Autenticación, login, token JWT, validación de sesión data/services/
MqttService Conexión MQTT, suscripción a tópicos, streaming de datos en tiempo real data/services/
CarsService Gestión de vehículos, datos de buses, información de flota data/services/
ReportsService Generación de reportes, exportación PDF/Excel, análisis de datos data/services/

4Comunicación MQTT en Tiempo Real

Configuración del Broker

Parámetro Valor

Puerto 1883

Características Principales

  • Conexión Automática: Intenta conectar automáticamente al iniciar
  • Reconexión: Reconecta automáticamente si se pierde la conexión
  • Suscripción a Tópicos: Recibe datos en tiempo real de ubicaciones
  • Stream de Datos: Proporciona un Stream para escuchar cambios
  • Manejo de Estados: Notifica conexión, desconexión y errores
⚠️ IMPORTANTE: El servicio MQTT está configurado como Provider en main.dart y se conecta automáticamente al iniciar la aplicación. Las credenciales están en auth en la clase MqttService.

7Consideraciones Importantes

1. Autenticación y Tokens JWT

  • La autenticación se maneja en AuthService
  • Los tokens JWT se validan con jwt_decoder
  • Se almacenan en Secure Storage (no en SharedPreferences)
  • Verificar expiración de tokens antes de hacer peticiones
  • Implementar refresh token si está disponible en el backend

2. Permisos Requeridos

Android & iOS necesitan los siguientes permisos:
  • 📍 GPS / Fine Location: Para localización en tiempo real
  • 📷 Cámara: Para capturar fotos y seleccionar imágenes
  • 💾 Almacenamiento: Para descargar reportes PDF/Excel
  • 🌐 Internet: Para comunicación con servidor

Usar permission_handler (^11.3.1) para solicitar permisos de forma elegante.

3. Conectividad

  • La aplicación requiere conectividad constante a internet
  • MQTT requiere conexión estable para actualizaciones en tiempo real
  • Usar connectivity_plus para validar conexión antes de peticiones
  • Implementar offline mode o caché si es necesario

4. Variables de Ambiente

Configuradas en: lib/data/global/environment.dart

  • URLs de APIs
  • Credenciales de servicios
  • Configuraciones específicas por entorno (dev, prod)

5. Compilación Android

  • Gradle: 8.11.1
  • Min SDK: API 21 (Android 5.0)
  • Requiere: Google Maps API Key configurada
  • Ubicación: android/app/build.gradle

6. API Keys Necesarias

  • Google Maps: Para visualizar mapas interactivos
  • Broker MQTT: satelitem2m.pe (usuario/contraseña en el código)
  • APIs Backend: Configuradas en environment.dart

Descripción del Flujo

  1. Loading Screen: Inicializa la aplicación y verifica si hay sesión activa
  2. Login Screen: Si no hay sesión, autentica al usuario con credenciales
  3. Home Screen: Dashboard principal con opciones de navegación
  4. Submódulos: Usuario puede navegar entre:
    • Map View: Ver ubicación de buses en tiempo real vía MQTT
    • Reports: Generar y descargar reportes PDF/Excel
    • History: Ver historial de movimientos de vehículos

9Checklist de Desarrollo

Antes de Empezar

  1. ☐ Instalar Flutter 3.13.4+ (verificar versión)
  2. ☐ Configurar Android SDK (Min API 21)
  3. ☐ Configurar iOS deployment target (si es para iOS)
  4. ☐ Configurar Google Maps API Key
  5. ☐ Obtener credenciales del broker MQTT
  6. ☐ Configurar URLs del backend en environment.dart

Configuración Inicial

  1. ☐ Clonar el repositorio
  2. ☐ Ejecutar flutter pub get
  3. ☐ Realizar flutter pub run build_runner build (si hay generated files)
  4. ☐ Configurar credenciales MQTT (en MqttService)
  5. ☐ Configurar variables de ambiente
  6. ☐ Compilar y ejecutar en emulador/dispositivo

Desarrollo

  1. ☐ Entender la arquitectura por capas
  2. ☐ Usar Provider para servicios compartidos
  3. ☐ Usar BLoC para lógica compleja (GPS)
  4. ☐ Mantener models en domain layer
  5. ☐ Usar go_router para navegación
  6. ☐ Agregar permisos en AndroidManifest.xml e Info.plist
  7. ☐ Testear GPS y MQTT en dispositivo real

Antes de Publicar

  1. ☐ Cambiar credenciales hardcodeadas por variables de ambiente
  2. ☐ Verificar que los permisos están solicitados correctamente
  3. ☐ Testear en diferentes dispositivos y versiones de Android/iOS
  4. ☐ Revisar logs y eliminar prints de debug
  5. ☐ Aumentar version en pubspec.yaml
  6. ☐ Generar APK/AAB para Android o IPA para iOS
  7. ☐ Testear instalación en dispositivo limpio
